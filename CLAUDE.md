# CDK Project

## 絶対ルール

- デプロイ・削除・push 等、外部に影響する操作は絶対に勝手に実行しない
- 操作の説明と実行確認を混ぜない。必ず分けて、明示的に確認を取る
- 選択肢を提示した場合、どれに対する承認か明確になるまで行動しない
- メモリへの書き込みも確認を取る
- 実装・デプロイ・検証・コマンド実行（参照以外）を行う前に、必ずこの「絶対ルール」を読み返すこと

## Push前チェック

- クレデンシャル情報（API key, secret, token, password, .env）が含まれていないこと
- .gitignore に不足がないこと
- `cdk.out/` がコミット対象に入っていないこと
