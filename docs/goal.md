# プロジェクトゴール

## 目的

このリポジトリは、既存の別環境を **AWS CDK ベースの新環境として作り直すプロトタイプ** である。

**第一目標は「既存環境と同等の動作を再現すること」**。

### スコープの考え方

- **機能・動作の再現**: 必須。既存環境にある機能はすべて揃える
- **既存にない機能の追加**: 原則しない（再現完了まではスコープ外）
- **技術スタックの置き換え**: 歓迎。既存と違うライブラリ・サービスでも、同じ動作が再現できるならむしろ積極的に提案して良い
  - 例: 「既存は X を使っているが、新環境では Y の方が適している」は OK
  - 例: 「既存にない Z を足すと便利」は再現完了後に別途議論

## 既存環境の構成

### バックエンド
- **GraphQL**: `graphql-yoga` + `@pothos/core` でスキーマ定義
- **スキーマ自動型生成**: バックエンドを保存するたびに GraphQL スキーマから **genql** + **urql** 用の型を生成し、フロントエンドはその生成型で API を呼ぶ

### フロントエンド
- **Next.js**（**React SPA への置き換えも可能**な構成）
- GraphQL クライアントは **urql**、型は **genql** で自動生成したもの

### リアルタイム通信
- **LiveKit**: WebRTC ベースのビデオ / 音声。**webhook 連携**あり
- **AWS IoT Core (MQTT)**: ブラウザ ↔ サーバー間のメッセージング

### 動画処理
- **ECS**: LiveKit の映像ストリームを受けて**顔モザイク処理**を行う

### 認証
- **Cognito**: ユーザーログイン

### データストア
- **RDS**: リレーショナル DB
- **S3**: ファイルストレージ

## このドキュメントの役割

新しい機能の実装を検討するときは、**まずこのドキュメントに書かれている要素かを確認**する。
ここに書かれていない機能の追加は再現完了後に議論する。
技術スタックの置き換え（既存と違うライブラリ・サービスで同じ動作を実現する）は、再現フェーズ中でも提案・採用してよい。
